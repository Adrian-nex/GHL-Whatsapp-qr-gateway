RETROSPECTIVA - Sprint review

Qué hiciste bien?
- Subimos todo el código a GitHub para tener un lugar central y seguro.
- Configuramos integración continua (CI) para que cada cambio se pruebe automáticamente y evitar sorpresas.
- Documentamos los pasos principales (cómo ejecutar y probar), lo que facilitó la colaboración.

Qué puedes mejorar?
- Simplificar las instrucciones para que cualquier persona, aunque no sea técnica, pueda ejecutar el proyecto.
- Añadir una sección breve de resolución de problemas comunes y cómo reportarlos.
- Hacer visible el estado del CI (badge) para que cualquiera vea rápidamente si el proyecto está sano.

Qué puedes mejorar para próximos proyectos?
- Automatizar el despliegue para reducir trabajo manual y errores.
- Crear una plantilla de repositorio con CI y pasos básicos ya listos para reutilizar.
- Preparar una "guía rápida" orientada a usuarios no técnicos además de la documentación técnica.

Frases cortas para la reunión:
- "Guardamos todo en GitHub para tener control y acceso centralizado." 
- "El CI comprueba automáticamente los cambios y nos ahorra tiempo y errores." 
- "La documentación está, pero la haremos más simple para la próxima."
